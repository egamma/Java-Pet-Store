Developer Policy for petstore workspace

This document describes some of the rules, standards, and conventions that we expect developers of the bpcatalog workspace to follow.

Public CVS workspace

Note, this project is developed on java.net as a public project. Please be sure that comments etc are appropraite for public viewing.

Project Directory Structure and Build Conventions

For the directory structure and placement of files and source code in the cvs workspace, we follow the BluePrints Project conventions. You can look at other examples of existing apps in the workspace as a guide.

For the build conventions, you can look at other examples of existing apps in the workspace as a guide. We are writing these details up and will add them soon. The project conventions offer some advice on this topic, such as placement of build files etc.

Issue Tracker

Also see the Issue Tracker policy doc in this workspace.

Legal

Note that every file should include the copyright notice embedded at the top of the file. This includes  java  files, xml files, JSP pages etc.
To insert the copyright, please include a blurb at the top. This blurb will be used later to insert and expand  the copyright when we build a dist build for release.The blurb is different for different file types.

For Java files:
/* Copyright 2004-2005 Sun Microsystems, Inc.  All rights reserved.  You may not modify, use, reproduce, or distribute this software except in compliance with the terms of the License at: http://developer.sun.com/berkeley_license.html
$Id: DeveloperPolicy.html,v 1.1 2004/06/22 22:45:39 sean_brydon Exp $  */
Please note that the $Id: DeveloperPolicy.html,v 1.1 2004/06/22 22:45:39 sean_brydon Exp $ gets replaced by cvs with a username and date.

For xml files: (Place this right below the <?XML ....> header)

Target Platform

Please be sure all code runs on GlassFish, the Java EE 5 SDK. We should all be using the latest release(promoted build).  Please be sure to include all the J2EE deployment descriptors and keep code portable.

Releases

We will be tagging the workspace after each release.

Code Style

Please follow Code Convention for the Java Programming Language, and remember that your code is read by lots of other people, so try to make it readable. And try to include good javadocs.
