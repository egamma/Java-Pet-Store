Petstore 2.0 Technology Features


This document outlines some of the technical features demontsrated in petstore

Here are some of the technologies and issues covered. Note, this is just a partial brief list. Overall, the goal is to provide a reference app for blueprints for building a web application on Java EE 5.
  • Using Java Persistence in a web application. Designing your model tier. Packaging choices. Conventions such as naming conventions
  • Using transactions in a web app. When to choose JTA, local, defaults. Patterns for hiding the transaction demarcation code, maybe with a filter
  • Resource Injection best practices.
  • Managing session state in a web application
  • Using a set of JSF components to build a web application.
  • Designing a set of JSF components.
  • AJAX-enabling the client interface
  • Project Conventions
  • Using XML APIs for serving data in an AJAX interaction.
  • Accessing mash-up web services in an web application. Petstore will access Google maps, possibly paypay.
  • Consuming an RSS feed as part of mash up. (BluePrints RSS feed is live and will serve some data to web app)
  • Security issues. Designing a captcha service. Dealing with proxies and mashups. Handling JavaScript/AJAX security model with Java EE security model.
  • Patterns to clean up and refactor and organize design of app.
Here are some other technology feautres
This is an overview of some Pet store features that we currently cover. This also covers some of the problems and solutions we cover. 
  • AJAX-enabled UI
    • single page architecture
      • Example, seler multi-step from in single page
      • Browsing VLH pattern in a single page
    • not really just one page
  • Integrate RSS into a web app
    • News feed on web app
    • Consume an external RSS feed, and display it in your web app
  • Proxy style mashups
    • need to access an external resource on another web app, dont want ot use client directly
    • JS security model
    • introduce proxy
  • Validation to prevent malicious code from user
    • User contributed content is part of web 2.0, but sometimes users can update code fragment
    • for example, either JS code entered into a rich text editor, that is stored in DB, and then displayed in your web app UI, and does some bad things to your UI.
    • for example, SQL injection
    • introduce layers of validation that prevent this.
  • Using a Captcha for security.
    • User content can have graffitti or spam, need a way to at least prevent *automated* graffiti. So introduce a catcha.
    • Does not require user identity, so more social and friendly
  • Securing some resources
    • web 2.0 makes many resources easily available thru HTTP, but still some urls in your app you want to protect.
    • some resources are publicly available by any http response, dont have to click thru gui if know url. But other resources, like maybe your file upload service, you would like users to be froced to go thru your application flow, and require a session etc.
    • introduce a security filter to protect sertain resources
  • Custom Search
    • how to integrate lucene search into an application
    • have Java Persistence data tier but also need search and indexing etc that interacts with DB tables directly
