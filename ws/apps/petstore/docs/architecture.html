Petstore 2.0 Architecture


This document will outline some of the key choices made for the petstore application, including exploration of various subsystems.

Web application

We chose to make it a pure web application. This means that we did not use the EJB technology to create reusable components. The Web components directly accessed the objection relational layer built with Java Persistence.

Patterns to structure application

The petstore uses the Model-View-Controller (MVC) architectural pattern for overall structure. The controller is split in two: Web Controller and Ajax Controller.
